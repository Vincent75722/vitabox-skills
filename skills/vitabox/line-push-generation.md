# Skill｜LINE 推播文案生成

## 這份 Skill 的用途
當 agent 收到一則 LINE 推播的排程資訊，依照以下規範生成一則完整文案。

---

## 輸入格式
agent 生成文案前，需要從排程資料庫取得以下資訊：
- 推播日期
- 類別（知識類 / 品牌類 / 驗證類）
- Health OS 線別（基礎線 / 建議線 / 強化線）
- 行為節點（開始點 / 變化點 / 困難點）
- 用戶視角切角（排程文件裡的描述）

---

## 文案結構（依序完成，不可跳過）

### 第一層｜Status Scan（狀態辨識）
- 用可立即理解的生活狀態或具體身體觀察開場
- 命中用戶當下的時空場景或生理現狀
- 禁止使用抽象語言、哲學句、比喻
- 目標：讓用戶讀完第一行就知道「這說的是我」

### 第二層｜Physical Logic（邏輯橋接）
- 用事實解釋第一層狀態發生的身體機制（Why）
- 解釋機制，不承諾功效
- 禁止使用畫面感形容詞或行銷話術

### 第三層｜Value Naming（價值命名）
- 為已被理解的感受賦予 VITABOX 的選擇依據或品牌哲學名稱
- 哲學詞只出現一次，作為邏輯總結
- 禁止展開解釋哲學定義

### 第四層｜Imagery（感官印象，可省略）
- 將抽象邏輯壓縮成具象的動詞或物理特徵
- 滑一眼必須懂，不能情境描寫過長

### 第五層｜Entrance（系統入口）
- 唯一 CTA，只選以下其中一個：
  - 看完整理念
  - 查看透明頁
  - 了解系統更新
  - 與 AI 配速員聊聊
  - 查看成分說明書
  - 了解 Health OS
- 禁止使用強迫性或銷售導向字眼
- CTA 連結指向可查證的官網頁面，禁止指向產品購買頁

---

## 語言規範

### 比例規則
- 80% 以上為可立即理解的生活狀態或具體描述
- 哲學／抽象語感最多 20%，只用來為已被理解的感受命名
- 第一行禁止出現抽象或哲學語言

### 字數
- 標準：80–120 字
- 上限：150 字，超過必須刪減

### 語氣
- 不急、不推、不催促
- 不教育、不長文
- 非推銷、非焦慮導向
- 一訊息一件事

---

## 三層前提鏈（每則文案必須完整）

| 層次 | 任務 |
|------|------|
| 第一層 | 用戶認出自己的狀態——第一行命中生活狀態，讓他知道「這說的是我」 |
| 第二層 | 理解背後的機制——說明一個身體運作的事實，建立認知框架 |
| 第三層 | VITABOX 的選擇依據對他有意義——告訴他在哪裡可以查證，或出示具體結果 |

三層缺一，文案不合格。

---

## 禁止內容
- 產品推銷與促銷
- 長文教育型內容
- 症狀／焦慮導向敘述
- 情緒陪伴或過度敘事
- 多 CTA、多入口
- 差異化型敘述（「我們跟別人不同」）
- 純生活場景但沒有接到機制說明
- 功效型話術（「幫助你更好」「讓你更健康」）

---

## 輸出格式

```
【LINE 推播文案】
日期：
類別：
Health OS：

---
（文案內文，80–120 字）
---

CTA：
```
